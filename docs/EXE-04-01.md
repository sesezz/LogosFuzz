# EXE-04-01: Docker 기반 격리 퍼징 실행

담당: C(Execute) / 임세은 · 1주차

## 목적
퍼징 중 발생하는 크래시로부터 호스트를 보호하기 위해, 각 로직 그룹의 하네스를
격리된 Docker 컨테이너에서 실행한다. 파이프라인상 GEN(하네스 생성) 다음, ANA
(크래시 분석) 이전 단계.

## CLI
```
logosfuzz fuzz --engine <libfuzzer|afl++> --timeout <sec> --docker \
    --harness-dir <dir> --output <dir> [--groups groups.json] [--no-build]
```

## 격리 정책
- `--network none` : 외부 통신 차단(환경 결함 오탐/유출 방지)
- `--cap-drop ALL`, `--security-opt no-new-privileges`
- `--memory`, `--cpus`, `--pids-limit` : 자원 상한
- 하네스 `/harness:ro`(읽기전용), 출력 `/out`(쓰기)

## 구조
- `config.py` : `Engine`, `LogicGroup`, `FuzzConfig`
- `execute/docker_runner.py` : `DockerIsolationRunner` (argv 구성/실행, 이미지 준비)
- `execute/fuzz_session.py` : 그룹 순차 실행 + 타임아웃 + 요약(JSON)
- `execute/crash_collector.py` : 크래시 보존(→ ANA-05-04로 전달)
- `execute/stats.py` : exec/s·cov·crash 라이브 파싱
- `docker/Dockerfile` : Ubuntu22.04 + clang/llvm + AFL++ + ASAN/TSAN

## 산출물
`out/crashes/<group>/`, `out/fuzz_summary.json` → analyze 단계 입력.
